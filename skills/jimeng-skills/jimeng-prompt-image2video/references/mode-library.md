# Mode Library — 图生视频 4 种子模式模板库

> 每种子模式的提示词结构、Keep/Move 模板和关键参数。

---

## 子模式 1：单图生视频 (Image2Video)

**输入**：1 张参考图
**核心逻辑**：从这张静态画面开始，描述接下来会发生什么运动

### 提示词公式

```
[从参考图画面开始] + [哪些元素在动 + 怎么动] + [哪些元素保持静止] + [运镜描述] + [时长] + [保持参考图风格]
```

### Keep/Move 模板

```
运动元素:[元素A] [运动方式+幅度+速度]；[元素B] [运动方式+幅度+速度]
静止元素: [元素C]、[元素D]、[元素E] 保持完全静止
运镜: [运镜类型 + 详细运动逻辑描述]
时长: [4-15秒]
风格: 保持参考图的风格和色调完全一致
```

### 关键原则

- **不描述画面已有内容**：参考图已经展示了主体、场景、色彩，提示词只需写运动
- **运动要具体到元素级别**："她的头发在微风中轻轻飘动" 而非 "有风"
- **静止声明必不可少**：AI 默认会动一切，不声明就会得到过度运动的画面

### 最适合的场景

风景微动化、人物肖像活起来、产品360度展示、静态画作动画化、光影变化

## 子模式 2：首尾帧 (Frames2Video)

**输入**：2 张参考图（第一帧 + 最后一帧）
**核心逻辑**：描述从图A到图B的过渡过程和中间发生了什么

### 提示词公式

```
[从图A的状态] + [过渡到图B的状态] + [过渡方式/节奏] + [运镜描述] + [时长]
```

### 过渡模板

```
起始画面: 图A的内容状态
结束画面: 图B的内容状态
过渡过程: [如何从A变到B - 渐变/生长/变形/季节更替]
过渡节奏: [前半段缓慢积蓄 → 后半段自然过渡] 或 [匀速平滑过渡]
运镜: [可选 - 通常保持固定或轻微移动]
时长: [8-12秒推荐]
```

### 关键原则

- **过渡方式是核心**：不是"变成"，而是"如何变成"——花开是"花瓣从外向内逐层展开"
- **节奏控制过渡的自然感**：匀速过渡显得机械，变速过渡更自然
- **两张图的风格必须一致**：否则过渡会不自然

### 最适合的场景

花开花落、季节更替、日出日落、建筑建造过程、人物年龄变化、产品变形

## 子模式 3：多帧故事 (MultiFrame2Video)

**输入**：2-20 张关键帧（故事板）
**核心逻辑**：帧与帧之间的因果关系和时间流逝串联成完整叙事

### 提示词公式

```
[帧序列的叙事逻辑] + [帧间过渡描述] + [时间流逝感] + [运镜串联各帧] + [时长]
```

### 故事板模板

```
帧序列: [2-20张关键帧]
叙事逻辑: [帧1发生了什么] → [帧2发生了什么变化] → [帧3继续发展]
帧间关系: [因果关系：因为X所以Y] 或 [时间关系：从A时刻到B时刻]
运镜串联: [如何用镜头运动将各帧连接为连续叙事]
整体节奏: [起承转合的节奏描述]
时长: [10-15秒]
```

### 关键原则

- **帧间关系是灵魂**：不是简单的时间顺序，而是有因果/情感的叙事逻辑
- **放弃提示词控制，用帧来控制**：故事板模式中，关键帧的视觉信息比提示词更重要
- **节奏感来自帧密度**：关键帧密集=快节奏，稀疏=慢节奏

### 最适合的场景

AI短剧、品牌故事、多镜头叙事、产品使用演示、教学视频

---

## 子模式 4：全能参考 (MultiModal2Video)

**输入**：最多 9 张图 + 3 段视频 + 3 段音频
**核心逻辑**：综合各参考素材中的特定元素，生成为最终目标服务的视频

### 提示词公式

```
[图一提供什么] + [图二提供什么] + [视频/音频提供什么] + [如何综合合成] + [运镜/风格描述] + [时长]
```

### 综合参考模板

```
参考分工:
  图一: [提供人物面部特征和姿势]
  图二: [提供场景环境和色调]
  图三: [提供服装材质和风格]
  视频参考: [提供运动节奏和运镜风格]
  音频参考: [提供情绪和节奏基调]
综合合成: [以上元素的交互关系和在画面中的呈现]
整体风格: [各元素风格的统一方向]
时长: [8-12秒]
```

### 关键原则

- **每份参考必须分工明确**：含糊的"综合这些素材"产生不可预测的结果
- **视频参考提供运动范式**：这是全能参考独有的优势——用视频告诉AI"这样动"
- **音频参考影响节奏**：音乐的快慢直接影响视频的剪辑节奏
- **图+视频+音频是最强组合**：三者结合可达到单一模式无法实现的精确控制

### 最适合的场景

人物+场景分别控制、口型同步视频、风格迁移视频、复杂多元素合成

---

## 子模式选择决策树

```
用户有参考素材吗？
├── 1张图 → 想要什么效果？
│   ├── 让图里的东西动起来 → 子模式1：单图生视频
│   └── 从A状态变到B状态（需要2张图）→ 子模式2
├── 2张图 → 子模式2：首尾帧
├── 3-20张图 → 是连续故事吗？
│   ├── 是 → 子模式3：多帧故事
│   └── 否（不同角度/不同元素参考）→ 子模式4
└── 图+视频+音频混合 → 子模式4：全能参考
```
